StewardWorks — Cozy Hub

A "room" dashboard. Every object on the wall and desk is a clickable entry point into a section of the app. This document maps each object to a route and lists exactly what the backend must provide.

www.stewardworks.space Frontend: interactive HTML prototype
How to use this: The attached HTML file is the live, clickable design — open it in a browser to feel the interactions. This page is the spec behind it. The prototype fakes navigation with a small "you're entering…" pop-up; in production, a click just navigates to the route shown.
SECTION A

The screen model

The hub is a single-page experience with one "home" screen (the room) and a handful of full-screen views layered on top. Closing any view returns to the room.

HOME

OVERLAY

OVERLAY

OVERLAY

SECTION C

User vs Admin — the one rule that matters

EVERY USER

Sees the full room and every object except the small key on the wall. Profile, Library, Environmental Literacy, Community, Help Desk, the Monitor programs, Meditation, Progress, and Log Out are all available.

ADMIN ONLY

Additionally sees the brass key on the wall → /admin (user management & settings). The key is not rendered at all for non-admins — don't just hide it client-side, gate it on the role.

Backend contract: the user session carries one boolean — is_admin (or a role). The frontend reads it to decide whether to render the key. The /admin route itself must also be protected server-side — never trust the hidden key alone.
SECTION D

What the backend needs to supply

AUTH
Login & session
A login page at /login. Log Out clears the session and returns there. Session exposes the user identity + is_admin.
PROFILE
User profile
Photo (shown in the wall portrait), learner type, dream job, and saved settings. Powers /hub/my-profile.
PROGRESS
Progress value (0–100)
One number per user. Drives the progress bar and the height of the chia sprouts on the desk statue. Updated as the user completes milestones.
LIBRARY
Saved & generated items
A list per user, each item: type (bookmark / generation / note), title, and source. Shown on the Progress page.
ADMIN
User management
Behind /admin: list/manage users and global settings. Role-gated server-side.
PAGES
The content routes
Each remaining object just needs its page to exist (Library, Environmental Literacy, Community Listening, Help Desk, the 3 Monitor programs). The hub only links to them.
SECTION E

Prototype-only — safe to ignore

These exist to make the demo feel alive. They are not backend features.

  • The day / dusk / night toggle and the color-cycling lamp — purely ambient visuals.
  • The "you're entering…" pop-up — a demo interstitial; real clicks just navigate.
  • Local browser storage (sw_progress, sw_timeofday) — fake persistence for the demo; replace with real user data.
  • The meditation tone & timer run entirely in the browser — no server needed.
